# Claims discipline for the measured explorables

These pages (`simulator/`, `sft/`, `preferences/`, and future beats) put real model
data in front of newcomers. Their whole value is that the numbers and claims are
**true and traceable** — the moment a reader catches one bluff, the "no faking" promise
that makes them worth more than a blog post is gone. This file is the standing rule for
keeping that promise. Read it before writing copy for one of these pages.

## The one rule

> **The page may say less than the artifact supports, never more.**

(Inherited from `../alignment/empirical/EVIDENCE_CONTRACT.md`, which governs the
empirical work behind these pages.)

## Two layers of enforcement

1. **Numbers → automated.** `audit-provenance.js` (run: `npm run audit:provenance`, or
   `npm run test:series` for pages+gate) drives every stage and fails if any number a
   reader sees traces to neither the page's bake nor a declared `<meta name="cited-number">`
   tag. It is proven to fail on a planted stat. Adding a citation forces you to name a
   source you actually checked.
2. **Prose → human.** The gate caught **0 of the 3** real violations this series shipped,
   because all three lived in wording, not digits. Before calling a page done, do a
   **full adversarial read of every rendered stage** and ask of each claim:
   - Is this a *law* I'm stating from *one* favorable example? (cherry-pick)
   - Does the artifact actually say this, or something weaker / different? (overreach)
   - Does what the reader *sees on screen* match what the copy *claims*? (self-refutation)

## The three we actually shipped (learn from these, they're concrete)

- **False fact.** SFT stage said *"a human wrote each answer."* SmolTalk is mostly
  synthetic — its largest slice (400K rows) was generated by Llama-3.1-405B-Instruct, and
  Beat 2 *already* disclosed UltraFeedback's ratings came from GPT-4. Inconsistent and
  wrong. → Fix: state what the source says; the honest version ("a big model taught the
  small one") was the better story.
- **Cherry-pick.** Register stage stated a general law — *"it matches the register it's
  given"* — while baking the one topic (of four) that obeyed it. Re-tested at equal dose:
  it mirrored **1/4**. → Fix: show all four, label mirrored vs corrected, state the split.
- **Self-refutation.** Preference-install copy said the concise answers *"still finish,"*
  while a 26-step display cap truncated 2 of 3 on screen. The model finished; our render
  didn't. → Fix: cap the display above the actual answer length; verify each shown example
  ends.

Common shape of all three: **the page asserting more than the artifact backs.** Same sin
we caught codex committing. Having the discipline is not the same as running it.

## Practical habits

- **Prefer showing the receipts to asserting them.** "1 of 4 mirrored" lands because the
  other three are right there to click. Make provenance a feature, not a footnote.
- **Write tests that assert the honesty, not just the click.** The register suite fails if
  anyone re-asserts the general law or mislabels a correcting topic — the guard survives
  the next well-meaning edit.
- **A display cap is a claim.** If you truncate generated text, the truncation is
  something the reader sees; make sure it doesn't contradict the copy.
- **Never report a metric that moves for the wrong reason.** distinct-2 rises mechanically
  on shorter text, so it is not reported on any length-changing beat.
